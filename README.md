# Vehicle Valuation System

## Project Overview

Vehicle Valuation System is a full-stack web application that helps users estimate car prices, explore similar car recommendations, and view useful market statistics based on a CSV dataset.

The system is designed to make car valuation simple and data-driven. Users can enter vehicle details, and the application uses the available dataset to return an estimated price and related insights.

### Key Features

- **Car Price Prediction**
    - Estimates the expected price of a vehicle based on its details and similar records in the dataset.

- **Similar Car Recommendations**
    - Suggests cars that are close to the selected or entered vehicle specifications.

- **Statistics and Charts**
    - Displays data insights using charts, such as price distributions, brand statistics, and other useful summaries.

---

## System Architecture

The system follows a simple client-server architecture.

### End-to-End Flow

```text
User → React → API → Express → CSV → Response → UI
```

### How the Flow Works

1. The user interacts with the React frontend.
2. The frontend sends requests to the backend API.
3. The Express server receives the request.
4. The backend reads and processes data from the CSV dataset.
5. The backend returns the result as a JSON response.
6. The React frontend displays the result in the user interface.

## How It Works

### Recommendations

Recommendations are generated by comparing the user-selected vehicle details with the records in the CSV dataset.

The system looks for cars with similar attributes, such as:

- Brand
- Model
- Year
- Fuel type
- Transmission
- Mileage
- Price range

The closest matching records are returned as recommended similar cars.

### Price Prediction

Price prediction is data-driven and based on similarity and averages.

Instead of using a separate database or external machine learning service, the backend analyzes the CSV dataset and finds vehicles with similar characteristics.

The predicted price is estimated using prices from similar cars, usually by calculating an average or weighted average from matching records.

### Statistics

Statistics are calculated directly from the CSV dataset.

The backend processes the dataset to generate useful summaries, such as:

- Average prices
- Brand-based statistics
- Model-based statistics
- Price distributions
- Dataset counts and grouped values

These results are sent to the frontend and displayed using charts.

## How to Run the Project

### Backend

Open a terminal and navigate to the backend folder:

```bash
cd server
npm install
npm start
```

The backend server runs on:

```text
http://localhost:3000
```

### Frontend

Open another terminal and navigate to the frontend folder:

```bash
cd vehicle-valuation-react
npm install
npm start
```

The frontend usually runs on:

```text
http://localhost:3000
```

If port `3000` is already used by the backend, React may ask to run on another port, usually:

```text
http://localhost:3001
```

---

## Environment Variables

### Backend Environment

Create a `.env` file inside the `server/` folder:

```env
PORT=3000
```

### Frontend Environment

Create a `.env` file inside the `vehicle-valuation-react/` folder:

```env
REACT_APP_API_BASE_URL=http://localhost:3000/api
```

This variable tells the React app where the backend API is running.

## API Endpoints

### Health Check

```http
GET /api/health
```

Checks if the backend server is running.

### Metadata Options

```http
GET /api/meta/options
```

Returns available filter options such as brands, fuel types, transmissions, and other dataset values.

### Models Metadata

```http
GET /api/meta/models
```

Returns available car models from the dataset.

### Price Prediction

```http
POST /api/valuation/predict
```

Predicts the price of a car based on the provided vehicle details.

### Recommendations

```http
GET /api/recommendations
```

Returns similar car recommendations based on selected criteria.

### Statistics

```http
GET /api/statistics
```

Returns calculated statistics from the dataset for charts and summaries.

### Contact

```http
POST /api/contact
```

Handles contact form submissions from the frontend.
